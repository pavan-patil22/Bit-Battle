  • Rules:
    • •Browser Restrictions: Do not open any other browsers or external tools like ChatGPT ,DeepSeek and etc...
    • •Violation: If found opening external sources, they will disqualified from this round.
    • •Completion Rule: Even if you finish early Do not close the site until round officially ends.

Round 2: Glitch-Fix

  • •Duration: 40 minutes
  • •Total questions: 4
  • •Max Score: Each correctly executed code carries 10 points (Total: 40 points)

What is Glitch-Fix?

  • Glitch-Fix is an error-finding round where participants are given code snippets containing bugs or errors. The task is to identify and fix the issues so the code runs correctly. It tests your debugging skills and understanding of programming concepts in C or Java.

Rules:

  • • Language: Code will be in C or Java only.
  • • Compiler: Participants must use the compiler provided by the organizers.
  • • External Tools: Do not use any external tools (e.g., ChatGPT, DeepSeek, etc.).
  • • Violation: If any participant is found using external sources, they will be disqualified from this round.
  • • Scoring Rule: The code must run successfully to earn points. No partial marks will be awarded.
  • • Final Decision: The decision of the teachers and our volunteers is final, and no further discussions will be entertained.

Round 3: Debugging

• Total Duration: 10 minutes (hint) + 30 minutes (debugging)

Scoring:

  • • Hint Cracked: 20 points
  • • Each Question Solved: 10 points (Total: 60 points)
  • • Total Possible Points: 80 points

Stage 1: Finding the Correct Hint (20 Points)

  • • Program displayed on projector for all participants.
  • • Participants must debug the program without running it.
  • • The correct output of the program is the password (hint) needed to proceed.
  • • Time Limit: 10 minutes to crack hint.
  • • Scoring:
    • • If you Crack the hint within 10 min → 20 points
    • • If not cracked in time, the correct hint is provided (but no points).

Stage 2: Unlocking Secret Files (60 Points)

  • • Correct hint unlocks next page with download button and 6 questions
  • • Download contains ZIP with 6 password-locked Word files

Stage 3: Debugging and Unlocking

  • • Each question on the webpage corresponds to one locked Word file.
  • • The correct answer (output) of each question is the password for the corresponding Word file.
  • • Participants must debug the questions(without running it), find the correct outputs, and use them as passwords to unlock the Word files.

    • Scoring:

  • • Each unlocked Word file = 10 points
  • • Total for 6 questions = 60 points

Stage 4: Confirming Completion

  • • Each Word file contains a programming-related meme as a reward.
  • • Once a participant unlocks a Word file, they must show it to the event volunteers.
  • • The volunteers will either take a picture or write down the proof on a page.
  • • Volunteers will keep track of scores.

Round 4: Coding

  • • Duration: 1 hour
  • • Max Score: 50 points

Round Format

  • • One long coding question with sample output
  • • The goal is to write a program that can perform a given task.
  • • Participants can Choice C or Java language to write their code.

Scoring System (50 Points)

  • • Points will be awarded based on specific conditions, which will be announced during the event.
  • • No predefined scoring breakdown
